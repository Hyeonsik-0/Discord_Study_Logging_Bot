## 무엇을 배웠는가

### `.env` 와 Spring Boot
Spring Boot 는 기본적으로 `.env` 파일을 자동으로 읽지 않는다. spring-dotenv 같은 라이브러리를 추가하거나 IDE/Docker 측에서 환경변수를 주입하는 방식이 필요. 같은 `.env` 한 파일을 IDE 실행 / Docker Compose / gradle 모두가 공유할 수 있게 만든 것이 핵심.

### Git Author 와 GitHub Contribution
git의 author email 이 GitHub 계정에 등록된 이메일과 매칭되어야 contribution graph 에 잔디가 박힌다. 자동 생성된 `username@hostname.local` 형식은 매칭되지 않음. `git config --global user.email` 로 명시적 설정 필요.

### 문서 구조의 중복
한 파일에 모든 내용을 넣으면 다른 문서 (README, architecture.md) 와 충돌하기 시작한다. 주제별 분리 + 인덱스 (README, devlog/README) 로 관리하는 편이 유지보수에 유리.
