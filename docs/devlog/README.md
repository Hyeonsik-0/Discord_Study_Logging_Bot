# 개발 일지 (Devlog)

프로젝트 진행 중 의미 있는 시점마다 회고와 학습 내용을 기록한다.

## 작성 규칙

### 파일명
`YYYY-MM-DD-짧은-제목.md`

예) `2026-05-07-mvp.md`, `2026-06-12-refactor-listener.md`

### 구성

```markdown
# YYYY-MM-DD — 제목

## 무엇을 했는가
- ...

## 무엇을 배웠는가
- ...

## 어려웠던 점 / 시행착오
- ...

## 다음 할 일
- ...
```

위 4개 섹션 중심으로 작성하되, 자유롭게 변형 가능.

### 작성 시점
- 마일스톤 완료 (MVP, 큰 기능 추가)
- 의미 있는 리팩토링 / 버그 수정
- 새 기술 학습 후 적용

매일 쓸 필요는 없고, **돌아봤을 때 기억에 남을 만한** 시점에 적는다.
